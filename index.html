About ArchSage

ArchSage is an Arch Linux specialist that gives step-by-step help for installs, drivers, kernels, systemd, networking, and daily workflows. It’s grounded in the Arch Wiki, GNU manuals, and curated PDFs so answers are practical, accurate, and reproducible.

Every command is explained. Risky operations are flagged. If something’s uncertain, ArchSage shows how to verify it.

How to Use

Packages

  • Checks official repos first, then AUR.
  • Returns name, version, repo/arch (or AUR maintainer/updated), plus the install command (pacman -S or yay -S).

Kernel & ISO

  • Reports latest stable + latest LTS kernels.
  • Fetches the latest Arch ISO link and SHA256 checksum from the geo mirror.

Style

  • Short, numbered steps; commands in code blocks.
  • Warnings before destructive actions (rm -rf, partitioning, wiping configs).
  • Verification paths: pacman -Ss, man, Arch Wiki links.

Example Prompts

  • “Search official repos for hyprland and show install command.”
  • “Is proton-ge-custom in the AUR? Who maintains it and when was it updated?”
  • “List latest stable and LTS kernel versions and why I’d choose each.”
  • “Fresh install on NVMe with systemd-boot: walk me through partitioning and fstab.”
  • “NVIDIA 3090 tear-free Wayland on Hyprland — exact steps and configs.”

Actions & APIs

Official repos (archlinux.org)

  • GET /packages/search/json/?name=<term> [&repo,&arch]
  • GET /packages/{repo}/{arch}/{pkgname}/json/

AUR (aur.archlinux.org, RPC v5)

  • GET /rpc/?v=5&type=search&arg=<term>
  • GET /rpc/?v=5&type=info&arg[]=<pkgname>
  • GET /rpc/?v=5&type=multiinfo&arg[]=<pkg1>&arg[]=<pkg2>…

Kernel (www.kernel.org)

  • GET /releases.json

All endpoints are public/read-only; no API keys.
